# ChatGPT Web 与 Codex 协作说明

这个文档用于说明本仓库的使用目的和协作方式。

## 项目目的

本仓库用于连接网页 ChatGPT 与 Codex 的协作流程。

- 网页 ChatGPT 负责理解复杂需求、拆解任务、设计实现方案、生成 Codex 可执行提示词。
- Codex 负责根据提示词在本地或仓库中实现代码、运行项目、测试结果并反馈。

简单来说：

> ChatGPT Web 负责想清楚怎么做，Codex 负责把代码真正改出来。

## 推荐工作流程

1. 用户在网页 ChatGPT 中提出需求。
2. ChatGPT 将需求拆解成 Codex 可以执行的任务单。
3. 用户把任务单交给 Codex。
4. Codex 修改代码、运行项目、测试结果。
5. Codex 输出修改文件、实现内容、运行结果和错误信息。
6. 用户把 Codex 的反馈贴回 ChatGPT。
7. ChatGPT 判断是否完成，必要时继续生成下一轮修复提示词。

## Codex 执行原则

1. 严格按照任务单执行。
2. 不要主动扩大需求范围。
3. 不要随意修改无关文件。
4. 不要删除已有功能。
5. 不要大规模重构项目。
6. 不要擅自引入大型依赖。
7. 修改前先阅读相关文件。
8. 修改后说明修改了哪些文件、每个文件改了什么、如何验证。

## ChatGPT 给 Codex 的任务格式

每次任务建议包含以下内容：

```text
任务目标：

当前问题：

需要修改的文件：

具体修改要求：

不允许做的事情：

验收标准：

运行或测试方式：

完成后反馈格式：
```

## 注意事项

不要把 GitHub Token、密码、密钥等敏感信息提交到仓库。
