# Project Collaboration Rules

## 项目目标

本项目用于连接网页 ChatGPT 与本地 Codex 工作流。

网页 ChatGPT 负责复杂需求理解、方案设计、任务拆解和提示词生成。

Codex 负责根据明确提示词在本地仓库中实现代码、运行项目、测试结果，并反馈修改内容。

## Codex 工作原则

1. 严格按照用户或网页 ChatGPT 给出的任务执行。
2. 不要主动扩大需求范围。
3. 不要随意修改无关文件。
4. 修改前先阅读相关文件，理解现有结构。
5. 保持原有代码风格。
6. 如果需要新增文件，必须说明新增原因。
7. 如果发现需求不明确，优先根据现有项目结构做最小合理实现。
8. 修改完成后必须说明：
   - 修改了哪些文件；
   - 每个文件改了什么；
   - 如何运行或验证；
   - 是否还有未完成问题。

## ChatGPT Web 输出给 Codex 的任务格式

每次 Codex 收到任务时，应优先识别以下结构：

1. 任务目标
2. 当前问题
3. 需要修改的文件
4. 具体修改要求
5. 不允许做的事情
6. 验收标准
7. 运行或测试方式

## 禁止事项

1. 不要删除用户已有功能。
2. 不要随意重命名页面、目录、组件或接口。
3. 不要为了“优化”而大规模重构。
4. 不要擅自引入大型依赖。
5. 不要提交敏感信息、token、密码、密钥。
